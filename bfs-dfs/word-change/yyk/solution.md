# 문제 풀이

## 핵심 아이디어
- 최소 변환 횟수를 구해야 하므로 BFS를 사용한다.
- 큐에는 `[현재 단어, 변환 횟수]`를 저장한다.
- 현재 단어에서 `words`의 각 단어로 한 글자만 다른지 확인하고, 가능하면 다음 탐색 대상으로 큐에 넣는다.
- BFS는 가까운 단계부터 탐색하므로 `target`을 처음 만났을 때의 횟수가 최솟값이다.

---

## 풀이 흐름
1. `target`이 `words`에 없으면 변환 자체가 불가능하므로 바로 `0`을 반환한다.
2. 큐를 `[[begin, 0]]`으로 시작한다.
3. 큐에서 하나씩 꺼내 현재 단어와 변환 횟수를 확인한다.
4. 현재 단어가 `target`이면 현재 횟수를 반환한다.
5. 아직 방문하지 않은 단어들에 대해, 현재 단어와 비교해서 다른 글자 수를 센다.
6. 다른 글자 수가 정확히 1개라면 변환 가능하므로 방문 처리 후 큐에 넣는다.
7. 끝까지 `target`을 찾지 못하면 `0`을 반환한다.

## Edge Case
- `target`이 `words`에 없는 경우: 어떤 경로로도 도달할 수 없으므로 `0`
- 시작 단어에서 바로 변환 가능한 단어가 없는 경우: 큐 탐색이 종료되고 `0`
- 여러 경로가 있어도 BFS가 가장 먼저 찾은 `target`이 최소 단계

---

## 시간 / 공간 복잡도

### Time Complexity
- `O(N^2 * L)`
- `N`: 단어 개수, `L`: 단어 길이
- 각 단어를 꺼낼 때마다 모든 단어를 비교하고, 비교할 때 단어 길이만큼 확인한다.

### Space Complexity
- `O(N)`
- 큐와 방문 리스트에 최대 단어 개수만큼 저장된다.

## 코드 메모
- 현재 구현은 큐를 리스트로 두고 `pop(0)`을 사용한다.
- 입력 크기가 작아서 동작에는 문제 없지만, 일반적으로는 `collections.deque`를 사용하면 더 효율적이다.
