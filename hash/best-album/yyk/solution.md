# 문제 풀이

## 🎯 접근 전략
- 이 코드는 베스트 앨범에 수록될 노래의 고유 번호를 결정하기 위해 여러 단계를 거칩니다.

1.  **데이터 구조화 (Grouping Data)**
    *   `album`이라는 빈 딕셔너리를 생성합니다.
    *   `genres`와 `plays` 리스트를 순회하면서 각 노래의 정보를 장르별로 묶습니다.
    *   각 장르(예: "classic", "pop")를 key로 사용합니다.
    *   value는 `total_plays`(해당 장르의 총재생 횟수)와 `songs`(노래 정보 리스트)를 담는 또 다른 딕셔너리가 됩니다.
    *   `songs` 리스트에는 각 노래의 `(재생 횟수, 고유 번호)`가 튜플 형태로 저장됩니다.

    ```python
    # 아래와 같은 구조로 데이터를 만듭니다.
    album = {
        "classic": {
            "total_plays": 1450,
            "songs": [(500, 0), (150, 2), (800, 3)]
        },
        "pop": {
            "total_plays": 3100,
            "songs": [(600, 1), (2500, 4)]
        }
    }
    ```

2.  **장르 정렬 (Sorting Genres)**
    *   `album` 딕셔너리를 `total_plays`를 기준으로 내림차순 정렬합니다.
    *   이를 통해 어떤 장르가 앨범에 먼저 수록되어야 하는지 순서를 결정합니다. (예: pop -> classic)

3.  **장르 내 노래 정렬 (Sorting Songs within Genre)**
    *   정렬된 각 장르에 대해, `songs` 리스트를 다음 두 가지 기준으로 정렬합니다.
        1.  노래의 재생 횟수(`plays`)를 기준으로 **내림차순** 정렬합니다.
        2.  재생 횟수가 같다면, 노래의 고유 번호(`index`)를 기준으로 **오름차순** 정렬합니다.
    *   `lambda x: (-x[0], x[1])` 키를 사용하여 이 두 가지 정렬을 한 번에 처리합니다.

4.  **최종 결과 도출 (Extracting Final Answer)**
    *   모든 정렬이 끝나면, 각 장르에서 가장 인기 있는 상위 2개의 노래만 선택합니다.
    *   선택된 노래들의 고유 번호(`index`)를 순서대로 `answer` 리스트에 추가합니다.
    *   최종적으로 이 `answer` 리스트를 반환합니다.
---

## ✅ 테스트 결과
- 정답률: 100%

## ⚠️ Edge Case
- 한 장르에 노래가 하나만 있을 경우, `[:2]` 슬라이싱을 통해 자동으로 해당 노래 하나만 선택되므로 별도의 처리가 필요 없습니다.

---

## 🕰️ 시간 / 공간 복잡도

### Time Complexity
- **데이터 구조화**: O(N), 여기서 N은 노래의 수입니다. 모든 노래를 한 번씩 순회합니다.
- **장르 정렬**: O(G log G), 여기서 G는 장르의 수입니다. 장르의 총 재생 수를 기준으로 정렬합니다.
- **노래 정렬**: O(N log K), 여기서 K는 한 장르에 속한 최대 노래 수입니다. 모든 노래에 대해 정렬이 수행됩니다. 최악의 경우(모든 노래가 한 장르인 경우) O(N log N)이 됩니다.
- **결과 도출**: O(G), 상위 2곡을 선택하기 위해 정렬된 장르를 순회합니다.
- **종합**: 전체 시간 복잡도는 정렬 단계가 지배적이므로 **O(N log N)** 입니다.

### Space Complexity
- **O(N)**: `album` 딕셔너리에 모든 노래의 정보를 저장해야 하므로, 노래 수(N)에 비례하는 공간이 필요합니다.
