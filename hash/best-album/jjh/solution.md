# 문제 풀이

## 🎯 접근 전략

1. sum: {장르: 횟수 합} <- 총 재생 횟수 많은 순 정렬(key=labda x: x[1], reverse=True)
2. idx: {장르: [[고유번호, 횟수], [고유번호, 횟수], [고유번호, 횟수] ...]} <- 재생 횟수 많은 순 정렬
   - sum 처럼 한 번에 정렬 불가능 -> idx의 key번 반복 필요, value: 이중리스트
   - 장르 하나를 뽑아서 횟수 크기 순으로 정렬(d_idx[j].sort(key=lambda x: x[1], reverse=True))
3. 장르의 수: k = len(sum), 2k개의 곡을 선택해야 함
4. sum을 통해서 먼저 고를 장르를 선택(총 횟수 많은 순) -> 하나밖에 없으면 하나만 선택 + 횟수 동일 시, 인덱스 낮은 순
