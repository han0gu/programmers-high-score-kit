# 문제 풀이

## 🎯 접근 전략

0.  의상의 종류별로 한 가지만 선택 가능
    - 종류: 2, 이름: 각 1, 2
    1. 요소 1(이름의 개수 합):
       - 종류 1(이름 1)
       - 종류 2(이름 1)
       - 종류 2(이름 2)

    2. 요소 2(2\*1):
       - 종류 1(이름 1) + 종류 2(이름 1)
       - 종류 1(이름 1) + 종류 2(이름 2)

    =====
    - 종류: 3, 이름: 각 2, 3, 4
    1. 요소 1
       - 종류 1(이름 1)
       - 종류 1(이름 2)

       - 종류 2(이름 1)
       - 종류 2(이름 2)
       - 종류 2(이름 3)

       - 종류 3(이름 1)
       - 종류 3(이름 2)
       - 종류 3(이름 3)
       - 종류 3(이름 4)

    2. 요소 2
       - 종류 1(이름 1) + 종류 2(이름 1)
       - 종류 1(이름 1) + 종류 2(이름 2)
       - 종류 1(이름 1) + 종류 2(이름 3)
       - 종류 1(이름 1) + 종류 3(이름 1)
       - 종류 1(이름 1) + 종류 3(이름 2)
       - 종류 1(이름 1) + 종류 3(이름 3)
       - 종류 1(이름 1) + 종류 3(이름 4)

       - 종류 1(이름 2) + 종류 2(이름 1)
       - 종류 1(이름 2) + 종류 2(이름 2)
       - 종류 1(이름 2) + 종류 2(이름 3)
       - 종류 1(이름 2) + 종류 3(이름 1)
       - 종류 1(이름 2) + 종류 3(이름 2)
       - 종류 1(이름 2) + 종류 3(이름 3)
       - 종류 1(이름 2) + 종류 3(이름 4)
         ...
       - 종류 1(이름 1): 7
       - 종류 1(이름 2): 7

    3. 요소 3
       -> 생성되는 요소의 최대 개수 == 종류의 개수

1.  딕셔너리 생성 -> {키(의상의 종류:str) : 밸류(의상의 이름:list(str))}
2.  경우의 수로 접근,
    1.  1 종류면 -> 의상 갯수
    2.  2 종류 이상이면, 각 의상 갯수 + 1 -> 각 요소를 모두 곱한 후 모두 안 입는 경우 -1
