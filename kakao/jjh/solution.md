# 문제 풀이

## 🎯 접근 전략

- 최소 공배수까지의 노란불 누적합 리스트 생성(get_lcm, get_cumsum 함수)
  - 노란불이 1초면 그냥 누적합 리스트 사용
  - 2초 이상인 경우, 생성한 누적합 리스트 복사 + 슬라이싱으로 사용할 노란불 누적합 리스트 자르기 때문에 정렬
    - 복사한 리스트에 2초 이상인 범위 추가
  - 슬라이싱을 통한 필요한 노란불 누적합 리스트 구하기
- 중복된 것이 있을 수도 있기 때문에 중복 검사 및 최소값 찾기(solution 함수)

## ⚠️ Edge Case

- [[1,1,1], [1,2,1], [2,1,1]] <- 노란불 기준으로 정렬하지 않으면 슬라이싱 범위 바깥에 필요한 누적합 리스트 생성
